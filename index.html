CL-PLO Decision Spine

Keep the portfolio model adaptive without forgetting how markets break.

Problem

Market behavior shifts. A model tuned only to recent flow can forget stress behavior.

Why It Matters

When stress memory fades, live allocations can look fine until a shock hits and drawdowns jump.

Core Idea

Keep the transformer fixed, and improve only the update rule with replay and a risk-feasible projection.

ANaive: adapts fast, but stress memory decays.

BReplay: keeps stress fixtures active in each update.

CHybrid: replay plus projected update inside risk limits.

Decision signal: choose the updater that adapts, keeps stress retention, and stays within risk limits.

1. Three Strategy Mechanisms

A live portfolio model has one hard requirement: keep adapting to new flow without erasing behavior learned from stress windows. All three pipelines share the same transformer, same data stream, and same optimizer. Only the update rule changes.

\(x_t\): current LLM market batch (tokens plus returns)

\(A_t\): anchor fixture set (stress windows kept in memory)

\(g_{d,t}\): drift gradient from current flow

\(g_{a,t}\): anchor gradient from replay fixtures

\(w_t\): portfolio weights sent to execution

C · Proposed here

Replay + Projection Gate

The drift step is clipped to the feasible region before replay is added. The model keeps adapting, but the update stays inside the risk budget.

Concrete projection view

The model proposes a move. If that move breaks the risk budget, we keep the direction and trim the size until it fits.

Read in 5 seconds

Proposed movewhat the model wants to do from new flow.

Allowed regionposition changes that respect risk limits.

Safe moveclosest allowed update sent to execution.

Proposed here

Equation in plain words: take the model's proposed move and keep the closest version that stays inside the risk budget.

\(g_{d,t}\) proposed move from current flow · \(\mathcal{F}_t\) allowed region from risk limits · \(g_t^{\mathrm{safe}}\) move sent to execution.

\[ g_t^{\mathrm{safe}}=\arg\min_{g\in\mathcal{F}_t}\lVert g-g_{d,t}\rVert_2^2 \]

Operational meaning: keep the live signal direction, then clip size to the nearest risk-feasible trade.

3. Deploy Path

How this ships in practice: one small update, one strict check, one clear decision.

Go / no-go checks

  • Go: hybrid stays first and drawdown lift is still positive.
  • Hold: stress retention or stress Sharpe falls below replay.
  • Rollback: live turnover or drawdown moves outside the approved band.

Takeaway: treat updater changes like any other production release, with fixed checks and a fast rollback path.
