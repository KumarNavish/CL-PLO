Strategy-specific pipelines

Strategy A: Naive continual update

Drift batch \(\mathcal{D}_t^{\mathrm{drift}}\)
→
Gradient \(\mathbf{g}_{\mathrm{d}}=\nabla\mathcal{L}_{\mathrm{drift}}\)
→
Update adapter \(\theta_{t+1}=\theta_t-\eta\mathbf{g}_{\mathrm{d}}\)
→
Forecasts \(\hat{\mu}_t,\hat{\Sigma}_t\)
→
Weights \(\mathbf{w}_t^\star\)

Objective

\(\min_{\theta}\;\mathcal{L}_{\mathrm{drift}}\)

Constraint

None at gradient level

Implication

Fast adaptation, highest forgetting risk.

Strategy B: Anchor-regularized update

Drift batch \(\mathcal{D}_t^{\mathrm{drift}}\)
+
Anchor memory \(\mathcal{D}_t^{\mathrm{anchor}}\)
  • "2008 credit spread dislocation"
  • "2020 limit-down shock"
  • "2022 inflation surprise regime"
→
Weighted loss \(\mathcal{L}_{\mathrm{drift}}+\beta\mathcal{L}_{\mathrm{anchor}}\)
→
Update adapter \(\theta_{t+1}=\theta_t-\eta\nabla\mathcal{L}\)
→
Weights \(\mathbf{w}_t^\star\)

Objective

\(\min_{\theta}\;\mathcal{L}_{\mathrm{drift}}+\beta\mathcal{L}_{\mathrm{anchor}}\)

Constraint

Soft, via \(\beta\)

Implication

Better retention than naive, but conflict is not explicitly clipped.

Strategy C: Anchor + projection (proposed)

Drift gradient \(\mathbf{g}_{\mathrm{d}}\)
+
Anchor gradient \(\mathbf{g}_{\mathrm{a}}\)
  • Stress-memory snapshots from anchor set
  • Regime history: calm, inflation, liquidity shock
→
Projection gate

\(\tilde{\mathbf{g}}=\Pi_{\mathcal{C}_{\varepsilon}}(\mathbf{g}_{\mathrm{d}})\)

\(\mathcal{C}_{\varepsilon}=\{\mathbf{g}:\langle\mathbf{g},\mathbf{g}_{\mathrm{a}}\rangle\ge-\varepsilon\}\)

→
Safe update \(\theta_{t+1}=\theta_t-\eta\tilde{\mathbf{g}}\)
→
Weights \(\mathbf{w}_t^\star\)

Objective

\(\min_{\mathbf{g}}\tfrac{1}{2}\lVert\mathbf{g}-\mathbf{g}_{\mathrm{d}}\rVert_2^2\)

Constraint

\(\langle\mathbf{g},\mathbf{g}_{\mathrm{a}}\rangle\ge-\varepsilon\)

Implication

Retention is enforced as geometry, not only as penalty weight.
